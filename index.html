Introduction to Asymmetric Key Cryptography

Asymmetric key cryptography, also known as public-key cryptography, uses two different but mathematically related keys - a public key for encryption and a private key for decryption. This eliminates the key distribution problem faced in symmetric cryptography.

Different keys used for encryption and decryption

The security of asymmetric cryptography relies on mathematical problems that are easy to compute in one direction but difficult to reverse, such as factoring large numbers or computing discrete logarithms.

Key Characteristics

  • Key Pairs: Uses mathematically related public and private keys
  • Key Distribution: Public keys can be freely shared
  • Digital Signatures: Provides authentication and non-repudiation
  • Speed: Slower than symmetric cryptography
  • Usage: Ideal for key exchange, digital signatures, and authentication

RSA Algorithm

The RSA algorithm, named after its creators Rivest, Shamir, and Adleman, is the most widely used asymmetric cryptography system. It's based on the practical difficulty of factoring the product of two large prime numbers.

RSA Key Generation Process

Key Generation Steps:
1. Choose two large primes: p and q
2. Compute n = p × q
3. Compute φ(n) = (p-1) × (q-1)
4. Choose e such that 1 < e < φ(n) and gcd(e, φ(n)) = 1
5. Compute d = e⁻¹ mod φ(n)

Public Key: (e, n)
Private Key: (d, n)

Encryption Process

c = mᵉ mod n
Where:
c = ciphertext
m = plaintext message
e = public exponent
n = modulus

Anyone can encrypt using the public key, but only the private key holder can decrypt.

Decryption Process

m = cᵈ mod n
Where:
m = original message
c = ciphertext
d = private exponent
n = modulus

Decryption uses the private key to recover the original message from ciphertext.

Digital Signatures

Signature: s = H(m)ᵈ mod n
Verify: H(m) = sᵉ mod n

RSA can create digital signatures that provide authentication, integrity, and non-repudiation.

Applications in Internet Technology

Asymmetric cryptography enables secure communication and authentication across insecure networks like the internet:

SSL/TLS Certificates

RSA is used in SSL/TLS handshakes to establish secure connections between web browsers and servers, enabling HTTPS for secure web browsing.

Digital Certificates

Certificate Authorities use RSA to sign digital certificates that verify the identity of websites, organizations, and individuals.

Secure Email

Protocols like PGP and S/MIME use RSA for encrypting emails and verifying sender authenticity through digital signatures.

Code Signing

Software developers use RSA to sign their code, ensuring that users can verify the authenticity and integrity of downloaded software.

Real-World Security

In production systems, RSA is typically used with:

  • 2048-bit or 4096-bit keys for adequate security
  • Proper padding schemes like OAEP to prevent attacks
  • Hybrid cryptosystems combining RSA with symmetric encryption
  • Secure random number generation for key creation

Symmetric vs. Asymmetric Cryptography

Understanding when to use symmetric vs. asymmetric cryptography is crucial for designing secure systems.

Feature Symmetric Cryptography Asymmetric Cryptography
Keys Used Single shared key Key pair (public and private)
Speed Faster Slower
Key Distribution Challenging Easier (public keys can be shared)
Encryption/Decryption Same key for both Different keys for each operation
Digital Signatures Not natively supported Native support
Use Cases Bulk data encryption Key exchange, digital signatures, authentication
Examples AES, DES, Blowfish RSA, ECC, ElGamal

Hybrid Cryptosystems

Modern secure communication combines both approaches:

  • Asymmetric cryptography establishes secure session keys
  • Symmetric cryptography encrypts the actual data transmission
  • This combines the key management benefits of asymmetric with the speed of symmetric cryptography
  • Used in SSL/TLS, VPNs, and secure messaging applications
